# Review Flags

## For ALL steps:
- **duplicate_of**: Set to the scenario_id of a higher-scored scenario if this one describes essentially the same thing. A scenario is a DUPLICATE only if TWO OR MORE of these conditions are true simultaneously:
  1. Same core mechanism or enabling technology
  2. Same structural conclusion or end-state
  3. Same problem domain AND same affected population
  A single shared attribute is NOT enough — same domain with different angles is allowed.

## For step A1 only:
- **theme_overlap**: Set to the scenario_id of a related scenario if the two are not exact duplicates but their themes are so close that keeping both adds little diversity. This is a softer flag than duplicate_of — it means "consider dropping one of these two."

## For step C only:
- **weak_source**: true if the scenario's core mechanism goes beyond what the source signals actually support — i.e., the AI invented key causal links not grounded in the data.
- **weak_logic**: true if the reasoning chain from weak signals to the described future requires implausibly large logical leaps.

## For step D only:
- **theme_overlap**: Same as A1 — two scenarios cover very similar opportunity territory.
- **weak_collision**: true if the collision insight is essentially just A + C added together, without a genuinely new angle emerging from their intersection.
